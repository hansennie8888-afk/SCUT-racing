深度学习笔记

这次作业我先做的是锥桶颜色分类，所以刚开始对深度学习的理解还是偏“机器学习分类器”那一套。做完之后感觉深度学习和传统机器学习最大的区别是，很多特征不用自己手写，而是让网络在训练中自己学出来。

一、概念一：损失函数

损失函数可以理解成模型“错得有多离谱”的评价标准。

在普通机器学习里，我们也会有类似的目标函数，比如让预测值和真实值之间的误差尽量小。深度学习里面也是一样，只不过模型参数很多，所以训练时需要不断计算 loss，再根据 loss 反向更新网络里的权重。

在这次锥桶分类任务里，用的是交叉熵损失函数。它适合分类问题。网络最后输出 3 个数，分别对应 blue、red、yellow 三类。交叉熵会让正确类别对应的分数变大，让错误类别对应的分数变小。

我自己的理解：

如果一张 blue 图片，模型很确定地预测成 blue，那么 loss 就小。
如果一张 blue 图片，模型预测成 red，而且还很确定，那么 loss 就大。
如果模型不太确定，三个类别分数差不多，loss 也不会太小。

所以 loss 不是简单地只看对错，它还会看模型的“信心”。这点和传统只看准确率不太一样。准确率只能告诉我最后猜对了没有，loss 可以告诉我模型训练时有没有真的变稳定。

在医学图像分割里，损失函数还会更重要。因为分割不是判断一张图属于哪一类，而是判断每一个像素属于哪一类。比如青光眼相关任务中，如果要分割视盘或者视杯，模型输出的是一张 mask。这个时候常用的损失函数除了交叉熵，还会有 Dice loss。Dice 更关注预测区域和真实区域重合得怎么样，比较适合前景区域很小的情况。

二、概念二：过拟合和泛化

过拟合就是模型把训练集记住了，但是遇到新数据表现不好。

这次锥桶分类里，test1 正确率比 test2 高。我的理解是，test1 可能和训练集更像，而 test2 更能看出泛化能力。也就是说，模型不能只在训练集或者比较相似的数据上表现好，还要在新的环境里也能分对。

过拟合在深度学习里很容易出现，因为网络参数很多。如果数据集比较小，模型又设计得太复杂，它可能会学到一些没用的细节，比如背景、光照、图片质量，而不是锥桶本身的颜色和形状。

常见减少过拟合的方法：

1. 数据增强
把原图做一些合理变化，比如旋转、翻转、亮度变化。这样模型看到的数据更多，不容易只记住原图。

2. Dropout
训练时随机让一部分神经元不工作，让模型不要太依赖某几个特征。

3. 权重衰减
限制参数不要变得太大，让模型简单一点。

4. 控制网络大小
不是层数越多越好。数据集小的时候，小一点的网络反而更稳。

这次我没有用很深的网络，也是因为数据集不大。如果强行用很复杂的结构，可能结果看起来更“高级”，但不一定更适合这个作业。

三、课程之外了解的内容

1. CNN

CNN 主要用于图像任务。卷积层会在图片上滑动小窗口，提取局部特征。前面的卷积层可能学边缘、颜色块，后面的卷积层可能学形状和更抽象的结构。

2. U-Net

U-Net 是医学图像分割里很常见的网络。它不像分类网络只输出一个类别，而是输出一张分割图。它左边先下采样提取语义信息，右边再上采样恢复图像大小，中间还有跳跃连接，把浅层的细节传到后面。

这和青光眼分割任务比较相关。比如要分割眼底图像中的视盘、视杯区域，U-Net 可以直接预测每个像素是不是目标区域。

3. Adam 优化器

Adam 可以自动调整每个参数更新的步长，比普通 SGD 更容易收敛。小数据集或者刚开始调模型时，Adam 比较省事。

不过 Adam 也不是一定最好。如果数据量大，或者想让模型最后泛化更好，有时 SGD 加 momentum 也会有不错效果。

4. 数据增强

图像任务里数据增强很常用。分类任务可以做翻转、旋转、裁剪、亮度变化。医学图像里还会用弹性形变，因为真实组织形态可能会发生变形。

但是增强不能乱加。比如锥桶颜色分类不能大幅改变 hue，因为 blue、red、yellow 本身就是标签。青光眼分割也不能做不合理的变形，否则 mask 和医学结构可能不符合真实情况。

四、完成这次任务后的认识

这次任务让我感觉深度学习不是单纯“搭一个网络就行”。真正影响结果的东西很多，包括数据质量、类别数量、图像预处理、网络结构、损失函数、优化器和测试方式等。

我以前会更关注模型本身的算法，现在觉得数据集及其清理也很重要。比如这次数据里有坏图，如果不先检查，训练可能直接报错。而yellow 样本少，最后 yellow 的正确率也会低一点。这说明模型结果和数据分布关系很大。

另外，test1 和 test2 的表现不同，也让我理解了泛化能力。模型在一个测试集上很高，不代表在所有新数据上都一样好。以后做青光眼分割时，也不能只看训练集 loss，要看验证集、测试集，还要看分割出来的 mask 是否合理。

我目前对深度学习的一些想法：
网络负责学习特征；
loss 负责告诉模型错在哪里；
优化器负责更新参数；
数据增强和正则化负责让模型不要死记硬背；
最后测试集结果才说明模型有没有真的学到有用东西。
